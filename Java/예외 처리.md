- 예외 처리
	1. 예외가 발생한 곳에서 직접 예외를 처리 : **try - catch**
		- 예외가 발생할 만한 명령문을 try 안에 묶고 catch로 대처를 예비한다
		- 하나의 try에 대해 **catch가 여럿** 올 수 있다
			- 단, 예외 클래스끼리 포함 관계에 있을 때는 범위가 작은 순서대로 처리한다. 범위가 큰 예외가 먼저 처리되면 범위가 작은 예외마저 잡아버리기 때문에 작은 예외의 처리문에 도달하지 못한다.
		- 예외가 발생해서 catch문으로 들어가면 body의 나머지 부분은 동작하지 않는다.
		- 예외가 발생하든 정상적으로 작동하든 꼭 돌아가게 하고 싶은 명령문은 **finally**에 넣는다
		- 형식
		**try** {
			예외가 날 만한 명령문들
		} **catch** (Exception e : 예외자료형 변수명) {
			예외 나면 이렇게 해라
		}\[catch...\]
		**finally**{
			예외 발생 하든 안하든 실행할 명령문
		}
	2. 예외처리를 메소드를 호출하는 쪽으로 맡긴다 : **throws**
		- 메소드 안에서 예외가 발생할 때 예외처리를 메소드 호출하는 쪽으로 전파하기 위해 사용
		
		  - 형식
			- 메소드명 **throws** 예외명
	3. 강제로 예외를 발생시키는 키워드 : **throw** 
		- 형식
			- **throw** new 예외클래스명();
